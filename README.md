# ProjectS

## 🤝 협업 규칙 (필독!)
프로젝트의 안정적인 관리와 충돌 방지를 위해 아래 규칙을 반드시 준수해 주세요.

### 1. 브랜치(Branch) 관리
* **`main` 브랜치 직접 작업 절대 금지**: `main`은 언제나 실행 가능한 최종 버전만 유지합니다.
* **개인 브랜치에서 작업**: 새로운 기능을 만들거나 에셋을 추가할 때는 반드시 본인 이름이 들어간 브랜치를 생성하세요.
    * **규칙:** `develop-이름`
    * **예시:** `develop-janghwan`, `develop-taehoon`

### 2. 커밋 메시지 규칙 (Commit Convention)
* 커밋 로그를 쉽게 알아볼 수 있도록 말머리를 붙여주세요.
    * `[Feat]` : 새로운 기능 구현
    * `[Fix]` : 버그 및 오류 수정
    * `[Asset]` : 그래픽, 사운드 등 리소스 추가
    * `[Refactor]` : 코드 구조 개선 및 정리
    * *예시:* `[Feat] 소코반 블록 밀기 기본 로직 구현`

### 3. 작업 완료 및 공유 (Discord)
* **작업 시작 전:** 반드시 `main` 브랜치의 최신 상태를 내 브랜치에 반영(Pull/Merge)하고 시작합니다.
* 자신의 브랜치에서 작업이 완료되면 **커밋(Commit) 후 푸시(Push)**를 진행합니다.
* 푸시 후에는 **디스코드 채널**에 아래 내용을 공유해 주세요.
    * 📸 스크린샷 또는 영상 (작업 내용 확인용)
    * 📝 작업 내용 설명 (수정되거나 추가된 사항)

### 4. 코드 병합 (Merge)
* 공유 후에는 깃허브에서 `main` 방향으로 **Pull Request(PR)**를 생성합니다.
* 리드(또는 팀원)의 코드 리뷰와 승인이 완료된 후 `main`에 최종 병합됩니다.

## 📁 프로젝트 구조 안내
* **Assets/Animations**: 애니메이터 및 애니메이션 파일 관리
* **Assets/Prefabs**: 캐릭터, 함정, 아이템 등 모든 프리팹 관리
* **Assets/Scenes**: 메인 씬 및 개인 작업용 씬 관리
* **Assets/Scripts**: 모든 C# 스크립트 관리
* **Assets/Sprites**: 도트 에셋 및 이미지 리소스
* **Assets/Textures**: 원본 이미지 에셋 파일 관리 (임시)

## ⚠️ 주의사항 (중요!)
### 🔴 Scene 파일 관리
* **동일한 씬 파일을 동시에 수정하지 마세요.** (깃 충돌 해결이 매우 어렵습니다.)
* 개인 작업용 씬(`Scene_이름`)에서 작업 후 프리팹을 만든 뒤, 메인 씬에 배치하는 방식을 권장합니다.
* **예시:** `Scene_JangHwan`에서 작업 완료 -> 프리팹화 -> 메인 Scene에 배치

### 🔴 Meta 파일 및 에디터 설정
* 유니티 에디터를 통해 파일을 이동/삭제하여 `.meta` 파일이 유실되지 않도록 하세요. (윈도우 탐색기에서 파일만 지우면 에러 발생)
* 프로젝트 컨텍스트 유지를 위해 **Edit > Project Settings > Editor**에서 `Asset Serialization`이 **Force Text**로 되어 있는지 확인해 주세요.
